Configuration - XBMC
  • Permettre le contrôle de XBMC en HTTP et vérifier le port.
          - Système -> Paramètres -> services -> Serveur Web
             (ne mettez surtout pas le port 8080 si XBMC est sur la même machine que Sarah)

  • Ajouter de la musique dans la Médiathèque:
          - Ajouter une source
          - Clique droit sur la source => 'Ajouter à la médiathèque'

  • Vérifier l'accès à XBMC depuis votre navigateur: http://ip:port

  • Mettez de la musique sur XBMC pour la suite des essais...

  • (Facultatif) Pour pouvoir lire les playlist .M3U:
    Par défaut, Xbmc prends les fichiers .M3U pour des images!
    Il est nécessaire de supprimer cette affectation en rajoutant ces lignes
    <advancedsettings>
      <pictureextensions>
        <remove>.m3u</remove>
      </pictureextensions>
    </advancedsettings>
    dans le fichier de configuration advancedsettings.xml (Le fichier est a créer si besoin)
    plus d'info ICI et LA!

  • (Facultatif) Installer l'extension Radio dans Xbmc:
          - Musiques -> Extensions -> plus -> Radio (auteur: Tristan Fisher) -> Installer

  • (Facultatif) Installer le plugin YouTube dans Xbmc:
          - vidéo -> Extensions -> plus -> YouTube (auteur: TheCollective) -> Installer

Configuration - S.A.R.A.H.
  • Paramétrer le plugin avec l'IP et port de XBMC (musique et vidéo).
         - exemple:
             api_url_xbmc_music       127.0.0.1:8800
             api_url_xbmc_video        192.168.0.1:8800
             (il n'est pas obligatoire de configurer api_url_xbmc_autre1 et api_url_xbmc_autre2)
         - Si le xbmc gérant la musique et la vidéo est le même, remplissez les 2 champs avec les mêmes infos.
         - Si xbmc est installé sur la même machine que Sarah (Nodejs), préférez 127.0.0.1 à l'adresse local (le plugin sera plus réactif avec cette IP).
         - La notion xbmc musique/vidéo est un exemple, vous pouvez organiser ça comme vous le souhaitez. (pensez à modifier le xml)

  • Si la musique est toujours en cours sur XBMC, Faites un essai de mise en pause en cliquant ici: [Pause]. Si ce simple essai fonctionne, tout va bien! (sinon aller directement à 'Test de la configuration')

  • (Facultatif) Si vous souhaitez moins de log dans la console NodeJS:
          - éditez le fichier xbmc.js et mettez la variable infodebug à false.

  • (Facultatif) 2 xbmc supplémentaires ou Basculement entre xbmc:New
    Il est nécessaire que les xbmc aient le même contenu (médiathèque), ou encore mieux, synchroniser! (Cf Google)
    Cela permet de lire un film, une musique, une radio (...) sur un ou deux xbmc supplémentaires.
    Cela permet également de pouvoir, par exemple, jouer une musique sur le xbmc vidéo en ajoutant une phrase aux commandes classiques:
         - Sarah, Je veux écouter AC/DC (Joue la musique sur le xbmc musique)
         - Sarah, Je veux écouter AC/DC au salon (Joue la musique sur le xbmc video)
         - Sarah, Je veux écouter AC/DC dans la chambre (Joue la musique sur le xbmc autre1)
    Pour cela:
          - Paramétrer le plugin avec l'IP et port XBMC (autre1 et/ou autre2).
          - Modifier le fichier xbmc.XML (ligne 30/35 environs) pour renseigner la phrase à ajouter
             au commandes classique pour utiliser cet xbmc.

Addon Sarah and XBMC: Principe et Structure
  • Principe :

    Sarah est avertie dès que vous faites lecture, pause, re-lecture, stop d'un média (musique, film, série, streaming,..) ou que celui se termine sans en enchainer un autre.

    La dernière action de xbmc est indiqué dans le status par status.lastaction. New

    Le status est maintenant mis à jour lors du démarrage du serveur. New

    Un Status est ainsi créer dans le contexte de Sarah: SARAH.context.xbmc.status.

  • Structure et données du status (Json):

    SARAH.context.xbmc.status:{
    statusvideo : { status du Xbmc identifié comme "video" dans la config du plugin
    xbmc: true/false, true si cet XBMC a démarré, false si arrêté
    player: 'stop/play/pause',Lecture arrêtée, en cours, en pause
    episode: -1/1/2/...,-1 = ce n'est pas une série. Sinon, numéro de l'épisode
    file: 'X:\\fichier',Emplacement et nom du fichier
    label: 'nomdufichier',le nom du fichier
    season: -1/1/2/...,-1 = ce n'est pas une série. Sinon, numéro de la saison
    showtitle: 'nomdelasérie',vide si ce n'est pas une série ou Le nom de la série
    title: 'letitre',Le titre du film, de l'épisode ou du média (streaming, TV, web..)
    type: 'movie/episode/unknow' Film, série ou inconnu (streaming, TV, web..)
    },
    statusmusic:{ status du Xbmc identifié comme "music" dans la config du plugin
    xbmc: true/false, true si cet XBMC a démarré, false si arrêté
    player: 'stop/play/pause',Lecture arrêtée, en cours, en pause
    artist: ['nomdelartiste'],Attention ARRAY du/des Nom d'artiste
    album: 'nomdelalbum',Nom de l'album
    title: 'titredelachanson',Titre
    label: 'nomdufichier',le nom du fichier
    file: 'X:\\fichier',Emplacement et nom du fichier
    },
    statusmixed: 'stop/play/pause'Status global: Stop si les 2 Xbmc sont en stop, Play si au moins l'un est en lecture, pause dans les autres cas
    lastaction:
    New     		'dernière action'Dernière action reçue:
      'music_and_video_xbmc_started'/'music_xbmc_started'/'video_xbmc_started'
      'music_and_video_xbmc_ended' / 'music_xbmc_ended' / 'video_xbmc_ended'
      'music_and_video_database_updated'/music_database_updated'/'video_database_updated'
      'video_started'/'video_paused'/'video_resumed'/'video_ended'/'video_stopped'/
      'audio_started'/'audio_paused'/'audio_resumed'/'audio_ended'/'audio_stopped'/
    }
Addon Sarah and XBMC: Utilisation
  • Variable Context de Sarah:

    Ce status peut être interrogé par un autre plugin de Sarah via le context: SARAH.context.xbmc.status
    exemple: ne lire un nouveau mail que si il n'y a pas de musique en cours:
    if (SARAH.context.xbmc.status.statusmusic.player=='stop') { ... }

  • Règle IFTTT de Sarah:

    La commande du status venant de xbmc peut être 'intercepté' par un autre plugin via les règles IFTTT de Sarah
    Cf doc de Sarah sur les règles IFTTT
    exemple: allumer/éteindre la lumière si Lecture/Pause d'un film ou d'une série
    La dernière action de xbmc est indiqué dans le status par status.lastaction (Cf Structure du json) ce qui permet d'agir directement sur valeur de cette variable.

  • fonction Event de Sarah New

    Cette fonction permet d'avertir les autres plugins dès un changement de status de xbmc.
    La doc de Sarah à ce sujet : .../wiki/JavaScript-API#event
    Cela equivaut à l'utilisation des IFTTT mais ce programme directement dans le plugin dans lequel vous souhaiter interroger le status xbmc. La dernière action de xbmc est indiqué dans le status par status.lastaction (Cf Structure du json) ce qui permet d'agir directement sur valeur de cette variable.
    exemple: éteindre la lumière si mise en pause d'un film
    dans votre plugin:
      exports.init = function(SARAH){
            SARAH.listen('xbmc', function(data){
                if (data.lastaction=='video_paused') { votre code ici pour allumer la lumière}
                if (data.lastaction=='video_resumed') { votre code ici pour éteindre la lumière}
                if (data.lastaction=='video_ended') { Sarah.speak('Le film était bien?');}
                if (data.lastaction=='video_xbmc_ended') { Sarah.speak('bonne nuit!');}
            });
      }

Addon StreamCam pour XBMC: Installation New
  • L'installation de cet Addon sur XBMC n'est pas obligatoire.

  • Il permet:

    Il permet l'affichage du streaming d'une ou plusieurs caméras en simultanées sur xbmc
    Il est possible d'afficher ce streaming dans des fenetres sans arrêter la lecture du média en cours. La source doit être une image et non une vidéo.

  • Installation sur XBMC:

      - Système -> Extensions --(racine)--> Installer depuis un fichier zip
         Sélectionnez le fichier script.streamcam.(Frodo ou Gotham).ZIP joint au plugin --> OK
         L'extension s'installe
      - Système -> Extensions -> Extensions activées -> Images --> StreamCam doit être maintenant présent.
      - Redémarrer XBMC

  • Pour XBMC GOTHAM:

      - Système -> Extensions -> Extensions activées -> Images --> StreamCam -> Configurer.
        Vous pouvez personnaliser les tailles et positions des fenetres,
        les durée d'affichage, le temps de rafraichissement et la fermetre automatique.
Addon StreamCam pour XBMC: Utilisation New
  • 3 Paramètres indispensable:

    xbmc - Le xbmc cible: music ou video
    url - l'url de l'image de la caméra
    window - le numéro de fenêtre souhaité pour l'affichage
         de 1 à 4 : 4 fenetres de 320x240 situées sur la partie basse l'écran.
         de 5 à 6 : 2 fenetres de 640x480
         7 : une fenetres de 1280x720

  • 3 Paramètres optionnels: UNIQUEMENT POUR XBMC GOTHAM!

    interval - pour forcer un temps de rafraichissant different. en millisecondes.
    autoClose - pour forcer la fermeture automatique ou pas. 'true' ou 'false' .
    duration - pour forcer un temps d'affichage different. en millisecondes.

  • Commande Vocale:

    2 exemples sont présents dans xbmc.XML, à vous d'adapter pour votre configuration (xbmc ? , url ? , window?)
    Attention de ne pas mettre le symbol & dans l'url: le remplacer par &amp;

  • Appel via url:

    .../sarah/xbmc?xbmc=music_ou_video&action=StreamCam&url=url_image_cam&window=num_fenetre
    exemple:
        .../sarah/xbmc?xbmc=video&action=StreamCam&url=http://192.168.xxx.xxx/axis-cgi/jpg/image.cgi&window=7

  • Appel interne plugin :

    SARAH.call('xbmc', {
         'xbmc' : 'music_ou_video' ,
         'action' : 'StreamCam' ,
         'url' : 'url_image_cam' ,
         'window' : num_fenetre
         'interval' : temps_en_millisecondes
         'autoClose' : 'true' ou 'false'
         'duration' : temps_en_millisecondes
    }, function(options){ };

    Exemple:
    SARAH.call('xbmc', {
        'xbmc' : 'video' ,
        'action' : 'StreamCam' ,
        'url' : 'http://192.168.xxxx.xxx/axis-cgi/jpg/image.cgi' ,
        'window' : 7
        'duration' : 30000
    }, function(options){ };

Commandes Vocales - Infos sur les nouvelles fonctions New
  • Gestion des albums

    Il est difficilement possible ou conviviale de demander directement un album de la même manière qu'un artiste. J'ai donc opté pour la méthode suivante. On peut demander à Sarah un album de tel ou tel artiste. Sarah interroge xbmc puis, via sa fonction askme, va annoncer le nombre d'album pour l'artiste en question et énuméré les titres d'albums. Vous pourrez alors demander l'album de votre choix en répondant le numéro de l'album ( Le 1, Le 2,...) ou sa position (le 1er, le 2ème, le 3ème,...) ou en répondant aucun pour teminer là.
         - SARAH :
               il y a 3 albums de AC/DC
               1. Back in black
               2. Highway to hell
               3. Ballbreaker
               Lequel souhaites tu écouter?
         - Vous :
               Le premier
       (ou) Le un
       (ou) Le deuxième
       (ou) Le 2
       (ou) Le troizième
       (ou) Le 3
       (ou) aucun

    Sarah gère un maximum de 15 positions (de 'Le premier' à 'Le quinzième', pour plus il faut obligatoirement dire 'Le 16', 'Le 17',...

    Sarah mets automatiquement en pause la musique pour une meilleur compréhension de la réponse

    Les commandes concernées par la gestion des albums sont :
         - Sarah, je veux écouter un album de ##ARTISTE##
         - Sarah, je veux écouter un album de cet artiste

  • Sarah, je veux écouter un album de ##ARTISTE##

    Cf Gestion des albums
    Cette commande utilise les même données d'importation XML des artistes:

  • Sarah, je veux écouter un album cet artiste

    Cf Gestion des albums
    Sarah va utiliser l'artiste en cours de lecture pour vous proposer ses albums.

  • Sarah, Je veux écouter tous les albums de cet artiste

    Sarah va jouer toutes les chansons de l'artiste en cours de lecture.

Configuration - Erreurs ou Problèmes
  • "Je n'ai pas pu contacter le serveur" : Aucune réponse de XBMC (ip, port, autorisation http...?)

  • "Je n'ai pas pu executer l'action" : Réponse de XBMC mais incorrecte. (XML...?)

  • Pour un XBMC réagissant lentement (connexion wifi ou pc lent): New
    Il peut être utile d'augmenter le timeout des requètes vers xbmc.
    Dans xbmc.js augmenter la variable requesttimeout (3000 millisecondes par défaut).
    et mettre la variable bigdatabasetimetoadd à 70 environs( 0 millisecondes par défaut).
    Cette dernière risque d'augmenter le temps de défilement lors de la recherche d'un titre/artiste/film en mode xbmc

  • Si, lors de la demande d'album, Sarah ne vous laisse pas le temps de répondre: New
    Il peut être utile d'augmenter les temps liés à Askme:
    Le temps donné à Askme dépends du nombre d'albums (= Temps Base + Temps Album x Nombre d'albums).
    Dans xbmc.js modifier les variables:
    - Temps Base : timeaskmealbum (7000 millisecondes par défaut)
    - Temps Album: timeaskmeaddedperalbum (3000 millisecondes par défaut)

  • Désactiver Chromeless (affichage des albums): New
    Dans xbmc.js modifier la variable:
    - chromeless à false (true par défaut)

Mode XBMC - Activation/Désactivation
  • Le mode XBMC permet de controler/télécommander XBMC sans devoir répéter "Sarah".

  • Activation:
         - Sarah, active le mode xbmc (Sarah répond)

  • Désactivation:
         - Merci, ça sera tout Sarah (Sarah répond)
              Ou
         - Automatiquement après un temps sans nouvelle commande

         - Durant le 'mode XBMC', seuls les fichiers lazyxbmc.xml et xbmc_temp.xml sont traité par Sarah.
           Vos autres plugins ne sont plus opérationnels.

         - Sarah contrôle toujours de manière plus précise le 1er mot (qui n'est plus Sarah)
           Il faut donc bien prononcer le 1er mot (ou adapter le xml en fonction de vos préférences).

  • Le mode a été testé et développé avec le thème par défaut de XBMC: Confluence.
         - Il fonctionne aussi avec Aeon Nox.
         - Aucune garantie de fonctionnement avec les autres thèmes.

Mode XBMC - Fonctions avancées
  • Le scrolling
    Le scolling s'arrête automatiquement après avoir bouclé complètement ou après avoir recu une demande quelconque.
    Vous pouvez régler la vitesse en modifiant dans le XML: out.action.value (millisecondes)

  • Analyse automatique du contenu
    Chaque fois que vous naviguez dans un menu , le plugin créer le fichier xbmc_temp.xml qui contient les choix possibles (dans une limite de 1000 items).
    Il suffit alors de prononcer le titre/série/film/... de son choix et xbmc ce positionnera dessus:
    exemple: Affiche les séries ...OK...NCIS ...OK... Saison 8 ...OK... Ships in the night...OK
    Au lieu de dire le titre, vous pouvez aussi dire "épisode XX" pour une série ou "Piste XX" pour un album.
    Si un épisode est nommé S01E02 trucmuche ou 01x02 trucmuche, il faut alors dire: Saison 1, épisode 2 ou trucmuche.

  • Analyse du contenu manuelle:
    Vous pouver demander à Sarah : "Analyse le contenu" afin d'anlyser à nouveau le contenu de la navigation en cours
    Cela est utile dans 2 cas:
    - Si vous lancer un addon qui fait une recherche sur internet (radio/ youtube,...),xbmc tarde a afficher le contenu qui a été traité trop tôt par le plugin
    - Si vous avez navigué dans xbmc sans passer par les commandes vocales.

  • Importation des personalisations:
    Si vous utilisez la fonction d'importation automatique des genres/artistes/séries et que vous les avez personalisé, cette fonction est pour vous!
    En demandant "Importation des personnalisations" à Sarah, Le plugin analysera vos personnalisations dans xbmc.xml et tiendra compte de celles-ci pour l'analyse du contenu.
    Il est nécessaire d'effectuer cette opération que si vous avez modifié les personnalisations.
    (Cette personnalisation est sauvegardée dans le fichier personnalisation.json)
Mode XBMC - Paramétrages avancés
  • Avec un xbmc installé sur une machine distante:
    Il se peut que la mise à jour du contenu se fasse trop tôt si le temps de latence (ping) entre la machine nodejs et celle xbmc est trop important.
    Dans ce cas, vous pouvez augmenter dans le fichier xbmc.js la variable delay_before_control_distant(500 millisecondes par défaut)

  • Modification du timeout de désactivation automatique:
    Le timeout est défini dans le fichier Custom.ini à la racine de Sarah et concerne le mode Lazzy de tous les plugins.
           ctxTimeout=60000 (60s par défaut)

  • Si votre thème XBMC est different de Confluence (ou autre compatible):
    Il doit être possible d'adapté le thème en modifiant le code de xbmc.js
    Dans la fonction navigation_context_viewmode_info, il faut paramétrer les temp_data pour chaque format d'affichage (viewmode)
         - temp_data.way_normal est le sens normal de défilement (celui du scrolling et de la recherche)
         - temp_data.way_reverse est le sens inverse de défilement
         - temp_data.way2_normal est le sens normal de défilement pour le second axe d'un tableau
         - temp_data.way2_reverse est le sens inverse de défilement pour le second axe d'un tableau
         - temp_data.way_options est le sens pour atteindre le menu d'affichage
         - temp_data.way_optionsback est le sens pour quitter le menu d'affichage
         - temp_data.first_col=1 laisser à 1
         - temp_data.last_col= le nombre de colonne du mode d'affichage

    Si vous créer une configuration pour un thème... partagez la! ;-)

XML - Infos et Avertissements
  • Info sur la génération automatique et la personalisation:

    En cas de problème de reconnaisance d'artistes, de genre ou de série par Sarah, vous pouvez personnaliser le xml en coupant/collant les lignes générées souhaitées entre:
    <!-- Personalisation XXXXX -->
    et
    <!-- XXXXX FIN Personalisation-->
    puis en les modifiant à votre convenance! (par exemple: U2: You toux)
    A la prochaine génération automatique, votre personalisation des artistes, genres ou série sera conservée!

  • Les caractères spéciaux: New

    Les caractères spéciaux & < > " [ ] sont maintenant correctement gérés.
    Si des artistes, film, genres, (...) comportent ces caractères, ces derniers seront supprimés ( <INCONNU> deviendra INCONNU , U2[Live] deviendra U2LIVE, ...)
    Des erreurs restantes peuvent être signalées dans la page html qui résume les importations effectuées.
    Toutefois, il est conseillé de vérifier dans Log2console si xbmc.XML est correctement chargé par la grammaire Sarah après des importations.

XML - Génération automatique du XML
  • Mise à jour des artistes dans le XML: ../sarah/xbmc?action=xml_artist&xbmc=music

  • Mise à jour des genres dans le XML: ../sarah/xbmc?action=xml_genre&xbmc=music

  • Mise à jour des playlists musicale dans le XML: ../sarah/xbmc?action=xml_playlistmusic&xbmc=music

  • Mise à jour des playlists vidéo dans le XML: ../sarah/xbmc?action=xml_playlistvideo&xbmc=video

  • Mise à jour des séries dans le XML: ../sarah/xbmc?action=xml_serie&xbmc=video

  • Mise à jour des chaînes dans le XML: ../sarah/xbmc?action=xml_channel&xbmc=video

  • Mise à jour des fims dans le XML: ../sarah/xbmc?action=xml_film&xbmc=video
